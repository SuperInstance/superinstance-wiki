### Components
1. **PBFTNode** — Full 5-phase PBFT (Castro & Liskov, 1999): Request → Pre-prepare → Prepare → Commit → Reply. Tolerates `f < N/3` Byzantine nodes.
2. **SemanticBFTNode** — Confidence-weighted voting (WBFT 2025). EMA-tracked reputation, payload complexity adjustment, weighted quorum: Σ(confidence) ≥ threshold.
3. **View Change Protocol** — Timer-based leader crash recovery, deterministic primary rotation, NEW-VIEW propagation.
4. **QDArchive** — MAP-Elites behavior grid (Mouret & Clune, 2015). N-dimensional descriptor space. Coverage + QD-score metrics.
5. **CMAESEmitter** — Self-adaptive covariance matrix (Hansen & Ostermeier, 2001). Samples offspring, updates from elites.
6. **FleetBreederConsensus** — BFT-gated breeding: leader proposes → 2f+1 commit → immutable decision → offspring → archive update.
7. **FleetBFTNetwork** — In-memory simulation with fault injection (Byzantine, partitions, delays).
