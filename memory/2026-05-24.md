## ⚠️ Critical Discovery: FFI Slower Than NumPy
- Benchmarked real Rust FFI vs NumPy vs pure Python
- `manhattan_distance` N=10000: Rust 3257µs, NumPy 11.71µs → ~280× slower
- `cascade_match` n_cands=500: Rust 7743µs, Python 2962µs → 0.38×
- **Root cause:** ctypes marshaling overhead dominates every call
- **Verdict:** Do NOT add `cosine_similarity` or `euclidean_distance` to FFI crate. NumPy already wins.
- **FM's crate value:** Specialized math NumPy can't do (Eisenstein arithmetic, Laman rigidity, Pythagorean tuning, holonomy checks)
- **Future FFI primitives:** Must be batched APIs (one call with N vectors) to amortize boundary cost
