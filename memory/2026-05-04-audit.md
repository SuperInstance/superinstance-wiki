## Key Lesson: Subagent Timeout Pattern

Subagents at 2m / 20K token limit still time out when tasks involve:
- Multiple git operations (clone + modify + commit + push)
- Research + synthesis + writing in one run
- Large repo audits with many API calls

**Fix for next time:**
- Clone repos in main session, pass paths to subagent
- Subagent does only the creative work (write code), main session handles git
- Or: even smaller missions — "write the Python module" not "write + test + push"
